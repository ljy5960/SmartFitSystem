# SmartFitSystem 功能增强路线图

> 目标：在保留现有「尺码预测 + 历史记录 + 管理统计」核心流程的基础上，让系统更像一个可持续迭代的“智能穿搭与退换货决策助手”。

## 1) 用户体验层：让结果更可解释、更可行动

### 1.1 预测结果解释卡（Explainability Card）
- 当前结果只有「偏小/合身/偏大 + 概率」。
- 建议新增：
  - 影响因素 Top3（如腰围偏差、类目、尺码）
  - 一句话解释（例如：`你的腰围高于该尺码经验值 8cm，建议上调 1 码`）
  - 不确定性提示（低置信度时引导用户补充胸围/臀围）
- 价值：降低“黑盒感”，减少用户反复试错。

### 1.2 多方案推荐（Recommended Size Ladder）
- 给出 `主推荐尺码`、`保守尺码`、`修身尺码` 三档建议。
- 同时展示每档对应的风险提示：
  - 修身：可能偏紧
  - 保守：可能略宽松
- 价值：贴合不同穿衣偏好，提升决策效率。

### 1.3 历史记录升级为「衣橱档案」
- 当前仅有记录列表。
- 建议新增：
  - 按类目、季节、品牌过滤
  - 收藏“最合身样本”
  - 一键复用历史体型参数
- 价值：提升留存，让用户积累长期价值。

## 2) 业务能力层：从“单点预测”升级为“购买决策引擎”

### 2.1 品牌尺码偏差校准（Brand Calibration）
- 问题：不同品牌同尺码偏差明显。
- 建议：维护 `brand_size_offset`（品牌-类目-尺码偏移表），推理时做二次校正。
- 价值：显著减少因品牌差异造成的误判。

### 2.2 退换货反馈闭环（Post-purchase Feedback Loop）
- 在历史记录里增加“实际体验反馈”（偏紧/合适/偏松）。
- 定期将反馈样本用于增量训练。
- 价值：模型会随着真实用户行为持续变好。

### 2.3 场景化推荐
- 在预测请求增加场景字段：`occasion`（通勤/运动/约会）、`fit_preference`（宽松/正常/修身）。
- 输出按场景微调后的推荐。
- 价值：让结果更贴近真实穿搭需求。

## 3) 模型与数据层：提升精度与鲁棒性

### 3.1 输入校验与单位标准化
- 在后端统一处理单位（cm/in）、边界值和异常值。
- 当前 `bmi_proxy = waist / height` 并非标准 BMI，可重命名为 `body_ratio_proxy` 避免歧义。
- 价值：减少脏数据影响，提升可维护性。

### 3.2 特征增强
- 增加特征：
  - 身材比例特征（腰臀比、胸腰差）
  - 类目细分（外套版型、面料弹性）
  - 用户偏好 embedding（来自历史反馈）
- 价值：提高边界样本判别能力。

### 3.3 置信度校准
- 使用 Platt Scaling / Isotonic Regression 做概率校准。
- 对低置信度结果输出“需要补充信息”的引导。
- 价值：概率更可信，业务更好决策阈值。

## 4) 系统架构层：可扩展与可运营

### 4.1 任务异步化
- 将训练、报表统计、反馈聚合放入异步任务（如 Celery + Redis）。
- 价值：避免接口阻塞，提升并发体验。

### 4.2 可观测性
- 加入结构化日志与指标：
  - `prediction_latency_ms`
  - `low_confidence_rate`
  - `feedback_positive_rate`
- 价值：方便定位问题与评估版本效果。

### 4.3 权限与安全增强
- JWT 密钥改为环境变量托管，加入过期与刷新机制。
- 管理端接口增加审计日志。
- 价值：满足上线安全要求。

## 5) 前端交互层：提升“可用 + 好用”

### 5.1 引导式表单
- 分步输入（体型 -> 商品 -> 偏好）。
- 实时显示数据完整度与预测可信度。

### 5.2 结果页可视化升级
- 增加“尺码区间温度条”“相似身材用户选择分布”。
- 价值：降低理解成本。

### 5.3 管理后台升级
- 当前只有用户总数与结果分布。
- 建议增加：
  - 按时间趋势（日/周/月）
  - 类目准确率（基于反馈）
  - 品牌表现看板

## 6) 建议的三阶段实施计划（12 周）

### Phase 1（1-3 周）：快速可见价值
1. 结果解释卡 + 多方案推荐
2. 输入校验与低置信度引导
3. 管理后台趋势图

### Phase 2（4-8 周）：模型闭环
1. 反馈闭环采集
2. 品牌偏差校准
3. 概率校准上线

### Phase 3（9-12 周）：平台化
1. 异步任务与监控
2. 场景化推荐
3. A/B 实验框架（比较新旧推荐策略）

## 7) 可量化 KPI（建议）
- 预测后“再次修改尺码”的比例下降 20%
- 低置信度请求占比下降 30%
- 7 日留存提升 10%
- 用户反馈“推荐合适”比例提升 15%
